Recoverable Password Generator

One password to keep them all ...

With thanks to Chris Veness and his javascript implementation of the Block TEA cipher (© 2005-2012 Chris Veness).

The idea is, you have one very simple master password which will generate random (and hence strong) passwords for all your other sites. The generated passwords will not be stored anywhere (it's all very well having one of those encrypted password containers, but what happens if that gets hacked?), but will be simple to recover if you forget them.

The master password has to be simple enough to be easy to remember, but not so common that you forget what it is! You put that in the "Master" field below.

Next, to generate a password for, say, hotmail, just type in the name of the site in the second field. Click "Generate". Voila! You have a password that is impossible to guess, but easy to recover if you forget it! I plan to leave this webpage up, so you can get to it whenever you need to.

Do this for every site for which you need a password.

Some points:

  • If you have to change your hotmail password after this, just use hotmail1, hotmail2 etc to generate new ones. Successive passwords are very different, so even if someone finds out your password, they won't be able to guess the next one.
  • Always use lower-case characters everywhere in the "master" and "site" fields. This is because you don't want to have to remember if you used "ebay" or "eBay" later on.
  • Nothing is being sent over the internet on this page: the generation is does entirely on your computer.
